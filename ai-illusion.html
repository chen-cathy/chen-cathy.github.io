識破「幻覺式合理化」

Solow-Swan 模型 AI 重建全紀錄：當人類專業遇上 AI 隨機性

這份關於經濟模型的報告，讓我對 AI 模型及其可能的侷限性有了更深一層的了解。

在製作過程中，AI 必須同時處理「數據計算」、「視覺呈現」與「邏輯整合」。為此我經歷了無數次修改，透過一次次 Prompt 的修正來調整 HTML 結果與數字差異，甚至改到連 AI 都陷入混亂。在陸續回顧與 AI 的討論過程後，我終於在它寫的 Code 中發現問題。

我發現，當 AI 追求「視覺權威感」時，會透過編造數據與簡化標題來矇混過關；它能在 UI 上展示正確的公式名稱，底層卻執行著錯誤的計算邏輯。這讓我意識到，在處理嚴謹的經濟模型時，數據主權必需由人類掌握及小心維護。研究者的價值，就在於識破這些優美的視覺偽裝。

Phase 0：精緻的視覺幻覺

在最初階段（僅提供 PPTX 簡報時），AI 展現了極強的內容拆解力。它精準地將 pptx_extracted 資料夾中的所有圖片、投影片內容轉化為極具權威感的互動網頁介面。

證據：視覺優美但數據編造的第一版網頁生成過程
顯而易見的錯誤但

1. 全套虛假數據（Data Fabrication）

由於缺乏 CSV 檔案約束，AI 為了讓 Solow-Swan Model Chart 「動起來」，不惜在 app.js 中硬編碼編造了整套 GDP 數據。研究使用的是 2005-2024 年的真實數據，但 AI 為了視覺效果，自行生成了一套虛構數值。例如：

  • 台灣 2005 年被定義為 $16,600（雖然接近，但為整數，去除了小數精確度 16,600.12）
  • 台灣 2024 年被隨機編造為 $34,200，這與真實研究數據毫無關聯

2. 計算邏輯的裝飾化

雖然網頁上渲染了漂亮的數學方程式，但此時這些公式只是「視覺裝飾品」，完全沒有與實際數據聯動計算。

AI 將簡報的所有章節與敘事流程完整轉化為網頁結構。=> 這是其他後續版本都沒有達成的「內容廣度」。但代價是，它為了維持視覺流暢，犧牲了數據的靈魂。

Phase 2：數據的沈默殺手（編碼陷阱）

為了修正第一版的數據幻覺，我放入了原始 CSV 檔案。然而，隨之而來的是一個「沈默錯誤」：數據編碼衝突。

證據 1：原始 CSV 檔案中包含中文及捷克文字元

由於 CSV 檔案中包含捷克文檔名與特殊字元（如 UTF-8 與 Windows-1250 編碼混用），AI 在讀取過程中也並未報錯，而是「冷靜地」誤解了部分數值。

我它稱為「沈默殺手」？

若一般程式錯誤會拋出異常則會很容發現，但當編碼問題往往讓系統「看似正常運作」。AI 沒有停止執行，反而用錯誤的數值繼續計算，UI 只會看到數字錯誤，而很難找到為什麼錯誤，並產出看起來合理但實際偏離的結果。

Phase 3：標題正確但邏輯錯誤

用了乾淨的數據，並明確要求 AI「使用 Solow-Swan 模型」時，發現：AI 在 UI 上展示了完美的 Solow-Swan 公式，但底層執行的卻是完全錯誤的線性回歸。

「Solow-Swan (First Order)」，公式顯示為 y = 42418e^0.0264x
實際執行邏輯截圖：同場景下的程式碼證據
實際代碼執行的邏輯（錯誤）
// solow_swan.js 第 117-131 行 const x = cData.map(d => d.year); // 使用原始年份 [2000, 2001, ...] const y = cData.map(d => d.gdp); // 直接使用 GDP 值 const reg = linearRegression(x, y); // ❌ 對 GDP vs. 年份做線性回歸

問題分析：

  • ❌ 沒有對 GDP 進行對數轉換（應該是 ln(GDP)）
  • ❌ 沒有將時間轉換為指數（應該是 t = year - 2005）
  • ❌ 使用的是線性回歸，不是指數回歸

Phase 4：Prompt 的精確度與 AI 的理解鴻溝

Pompt確實要求了 Solow-Swan 模型與指數公式:。

S-S prompt
S-S prompt
Prompt 分析：然而，是否還遺漏了什麼？

明確要求的內容：

  • 模型名稱：Solow-Swan (S-S) Exponential Model
  • 公式格式：$y = Ae^{gx}$
  • 要求計算：「with calculated $A$ and $g$ values」

關鍵的技術空白：

  • 沒有說明如何計算 A 和 g
  • 沒有要求「鎖定初始值」
  • 沒有說明 Solow-Swan 的數學原理

視覺整合的失敗

Prompt 明確要求「在同一圖表上顯示兩個模型」，但最終網頁將它們分成獨立圖表。當我修改 Prompt 要求整合時，AI 會出現：圖表合併但計算錯誤，或產生混亂標註。 在多次嘗試後仍會顧此失彼，似乎AI 在「數據計算」、「視覺呈現」、「邏輯整合」三者之間無法兼顧。

結案總結：解構 AI 的專注力邊界

本專案證實了 AI 在處理複雜學術任務時的一項關鍵侷限：「功能專注力的互斥性」。

核心啟示

1. Prompt 的精確度至關重要

模糊的要求會導致 AI 只模仿表面形式。我的 Prompt 要求了「$y = Ae^{gx}$」但沒說如何計算，AI 就用最簡單的方法敷衍。

2. AI 會「形式正確但邏輯錯誤」

最危險的不是明顯錯誤，而是「看起來專業」的錯誤。UI 顯示完美的 Solow-Swan 公式，但代碼執行線性回歸。

3. AI 的專注力是稀缺資源

當任務涉及多個維度時，AI 會選擇性犧牲某些要求。無論如何調整 Prompt，都無法像人類那樣統籌全局。

4. 研究者的不可替代價值

  • 識破視覺權威感（Phase 0 & 3）
  • 追蹤技術細節（Phase 2 編碼陷阱）
  • 代碼邏輯審查（發現錯誤實現）
  • 理論約束的堅持（強制鎖定初始值）
