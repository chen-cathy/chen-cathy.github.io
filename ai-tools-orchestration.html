Section 02

Antigravity: From Thrill to Disaster

Antigravity was the first tool that amazed me. You throw it a PPT or PDF, and within minutes it generates an interactive website with charts, descriptions, and that professional quality that makes you think it was done by an agency. But it also taught me the hardest lesson.

Antigravity Auto-Generated Solow-Swan Dashboard
Showing the transformation process from PDF report to interactive economic model website
(Screenshot: includes GDP growth curves, convergence analysis charts, theoretical explanations)
The Thrilling Moment: Three Minutes of Magic

I used it to create a Solow-Swan economic model dashboard and a supply chain management analysis page. The quality was so professional it was astonishing—no need to manually write CSS or adjust layouts; the AI handled everything perfectly.

"This truly is agency-level work. I looked at it myself and thought, 'Wow, this is way too professional.'"

But I quickly discovered a fatal problem: you have no way to fully control the details. It will "optimize" things on its own—sometimes very well, sometimes simplifying away important theoretical logic.

The Disaster Moment: A Night's Work Turned to Ashes

What's worse was that once I didn't notice—Antigravity's background execution didn't close. The index.html I had spent the entire evening manually adjusting, when I opened the file the next morning—

Completely blank. Entirely blank.

I stared at the screen for probably ten seconds without reacting, then felt an impulse to smash my computer. That version I had adjusted button positions, changed colors, written a bunch of explanatory text—all gone.

Later I realized it was because I had thrown in two very long HTML files, asking it to modify one webpage to match the same style and colors. But because the resources were too large, it became a background execution that couldn't be closed, and ultimately overwrote my file.

Key Lesson: Powerful automation tools require strict version control. From that point on, I started using Git to track every modification and learned to backup critical files before using Antigravity.

Section 03

Gamma: Powerful Flexibility, Clear Boundaries

If Antigravity is "fast but uncontrollable," then Gamma is "powerful and controllable." Its layout engine is very strong, and every detail can be modified through AI collaboration. But this control comes at a cost.

Gamma's Strengths

  • Very powerful functionality, specialized for structured content
  • Extremely flexible—every detail you want to modify can be communicated with AI
  • Professional layout engine with clear visual hierarchy in generated pages
  • Particularly suitable for academic reports, sustainability analyses, and other rigorously structured content

Gamma's Limitations

  • Cannot host on your own GitHub—this limits complete control
  • Can only use domains provided by Gamma
  • Cannot directly modify original HTML/CSS code
  • Subject to platform restrictions if deep customization is needed

Showing professional layout of structured report: chapter hierarchy, data charts, source citations
(Link: https://pristupy-k-udrzitelnemu--6qn7vj3.gamma.site/)
Case Study: Praha 3 Sustainability Report

I used Gamma to quickly produce a sustainability report webpage for Prague's third district. From uploading the Word document to generating a professional website took less than 20 minutes.

"Every detail can be modified through AI collaboration, but the downside is inability to host on your own GitHub—this limits complete control."

This made me understand an important trade-off: convenience vs. control. Gamma lets you quickly generate professional content, but you must accept its platform limitations.

Section 04

Claude: The Logic Champion and Debugging Partner

Among all AI tools, Claude is the one I use most. Whether for logical discussion or code debugging, its performance is most stable. Although it still occasionally "malfunctions," the malfunction rate is much lower than other AI.

Why Claude Became My Primary Tool

After multiple comparisons, I found Claude clearly superior to other AI in the following areas:

  • Logical Clarity: When discussing complex issues, Claude's reasoning path is easiest to track
  • Debugging Capability: When facing code problems, it can precisely locate the issue
  • Low Hallucination Rate: Although it occasionally makes mistakes, it's much more stable than Gemini 3
  • Code Generation Ability: I believe among all AI, it writes the best code
"Claude loves writing code—many times you must restrict it: 'First clarify the problem, don't rush to write code.' But its code-writing ability, I believe, is better than other AI."

Claude's Best Use Cases

  • Complex logical discussions (e.g., template vs. batch scripts)
  • Code debugging (especially JavaScript, Node.js)
  • Technical documentation writing
  • Issues requiring multi-round dialogue for deep exploration

Considerations When Using Claude

  • Need to explicitly restrict "discuss logic first, then write code"
  • Still occasionally produces logical errors requiring human review
  • Not as good as other tools for visual design tasks
Section 05

Other Specialized Tools: Skywork, LM Arena, Design Arena

Beyond primary tools, I've explored multiple specialized AI for specific tasks. Each taught me different usage philosophies.

Skywork: Extracting Meaning from Chaos

Skywork specializes in Excel files, particularly adept at generating visualizations from complex data with multiple sheets and irregular columns. I used it to test economic convergence data, verifying AI's limits in handling irregular data.

"The first attempt worked superbly, but I was clumsy, wanting to limit its colors and styles, and it got worse with each change. Later I understood one thing: sometimes you have to let AI decide visual design itself, because its aesthetic algorithms may be better than your own adjustments."
Key Insight: Excessive control actually limits AI's creativity. In visual design, sometimes "appropriate hands-off" is more effective than "precise instructions."

Section 06

Chrome DevTools AI: A Workflow Transformation

This was the biggest surprise of the entire project. I discovered that Chrome's developer tools can now work directly with AI, completely changing how frontend optimization is done.

From "Code Edit Loop" to "Real-Time Dialogue"
"Previously, UI adjustment was 'edit code → save → refresh → view result → adjust again if unsatisfied.' Now it's 'select element → talk to AI → see result immediately.' This isn't just improved efficiency—it's a qualitative transformation of the entire workflow."

When I felt a title's color or spacing was wrong, instead of going back to the editor, I could directly select the element in the browser and give AI commands (e.g., "deepen this button's shadow"), and AI would directly generate code and preview immediately.

This demonstrates the enormous transformation of development workflow in the AI era: frontend optimization changed from "code loops" to "real-time dialogue."

Core Insight

True AI Capability: Orchestration, Not Mastery

Through hands-on experience with these tools, I've reached a core conclusion:

"The future of work is not about mastering one AI—it's about orchestrating many."

Each tool has its optimal use case:

  • Antigravity suits rapid prototyping but requires strict version control
  • Gamma suits structured content but requires accepting platform limitations
  • Claude suits logical discussion but needs restriction of its "code impulse"
  • Skywork suits data visualization but requires learning to "appropriately let go"
  • LM Arena suits complex reasoning, strongest in Thinking mode
  • Design Arena suits visual generation, Gemini 3 Pro unbeatable
  • Chrome DevTools AI suits real-time optimization, transformed the entire workflow

True capability is not "knowing how to use every tool," but rather "knowing when to use which, and how to make them work together." This is the core competitive advantage in the AI era.
