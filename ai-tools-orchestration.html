Section 02

Category 1: One-Click Website Generators

The common thread among these three tools is "input content, get a webpage," but each has its own unique characteristics and use cases.

Tool 1: Antigravity

Antigravity was the first tool that truly amazed me. You simply import a PPT or PDF, and in minutes it generates an interactive website complete with charts and descriptions—professional quality that feels like agency work. I used it to create the Solow-Swan Economic Model Dashboard and the Supply Chain Management Analysis page. The quality was professional, requiring zero manual CSS or layout adjustments; the AI handled it all perfectly.

Antigravity Auto-Generated Solow-Swan Dashboard
(Demonstrating the conversion from PDF report to interactive economic model website)

However, complex economic models required many iterations. Eventually, the AI got "tired." AI does get "tired"—its performance degrades. It tries to "optimize," sometimes successfully, but other times it oversimplifies and ignores key theoretical logic. Worse, I once forgot to stop Antigravity's background process. I spent an entire evening manually tweaking the `index.html`, but when I opened it the next morning, the file had been "eaten"—overwritten completely.

I realized this was because I had imported two very long HTML files simultaneously and asked the program to modify one to match the other's style. The resource demand was too high, causing my computer to lag, so I force-closed the interface but continued coding elsewhere. Unbeknownst to me, the background process was still running, and it eventually overwrote my work. It was a false alarm since I had backups, but it taught me a valuable lesson: be careful. Since then, I've used Git to track every change and always backup critical files before using Antigravity.

Tool 2 & 3: Gamma and Skywork

If Antigravity is "fast but uncontrollable," Gamma is "powerful and controllable." Its layout engine is robust, allowing for AI-collaborative edits on every detail. I used Gamma to quickly build the Prague District 3 Sustainability Report website—from uploading the Word doc to a professional site in under 20 minutes. Every detail—images, text, layout—could be tweaked via AI. This taught me a key trade-off: Convenience vs. Control. Gamma gives you speed and polish, but you accept platform limits—hosting on their domain and no direct code access.

Gamma-Generated Praha 3 Sustainability Report
(Showcasing the professional layout of the structured report: hierarchy, data charts, citations)

Skywork specializes in Excel files, excelling at visualizing complex data from multiple sheets and irregular fields. I used it to test economic convergence data, pushing the limits of AI handling irregular data. The initial results were great, but I made the mistake of trying to micromanage the colors and style, making it worse. I learned that sometimes you have to let the AI decide the visual design because its aesthetic algorithms might be better than your manual tweaks. Over-control can limit AI creativity. In visual design, sometimes "letting go" is more effective than "precise instruction".

Section 03

Category 2: Model Arena Tools

These tools allow you to compare different model capabilities or use specific specialized models. LM Arena is for reasoning; Design Arena is for image generation.

Tool 4 & 5: LM Arena and Design Arena

LM Arena is one of my most-used tools. It's a random battleground for AI models, but also allows you to select specific ones. I mainly use it for high-difficulty reasoning tasks, especially those requiring complex information processing like Criminal Law Case Analysis. The best model has been claude-opus-4-5-20251101-thinking-32k, where the "Thinking" mode reveals the full reasoning process.

But there's a distinct phenomenon: the quality of the generated webpages degrades over time. By the 2nd or 3rd revision, if the initial prompt wasn't perfect, fixing specific issues becomes difficult. It's often better to start a new chat window, which I usually do the next day. This "diminishing returns" issue taught me that for complex tasks, it's best to clarify requirements upfront rather than relying on endless revisions.

Design Arena isn't for generating websites, but images—similar to comparing model capabilities. I used it to draw illustrations for criminal law cases. Gemini 3 Pro consistently produced the best results, with image quality far surpassing other AIs. This clarified that different models have different strengths: Claude is supreme in logic and coding, while Gemini 3 Pro rules visual generation.

Section 04

Category 3: Integrated Development Environments

The feature of these tools is "embedding into existing workflows," enhancing AI capabilities directly within your environment without switching platforms.

Tool 6: VS Code with AI Extensions

One of VS Code’s most impressive features is its ability to generate files, manage project structure, and edit code simultaneously, all within a single integrated environment. When combined with AI, it significantly lowers the barrier to technical web development.

ILate in the project, when I needed to fine-tune details, the VS Code + AI workflow was strikingly effective. I used it to translate an entire page while automatically generating a multilingual index at the same time. With GDPR considerations in mind, it was even able to generate legally compliant pages and interaction requirements without the need for highly detailed prompts.

This is not merely an improvement in efficiency; it represents a qualitative shift—from “editing files one by one” to “systematic project management.” For projects that require maintaining multiple interrelated files, this kind of integration is essentially irreplaceable.


(Demonstrating multi-file project management: simultaneous editing, link fixing, batch translation)

Tool 7: Chrome DevTools AI

This was the biggest surprise of the project. I discovered that Chrome DevTools can now collaborate directly with AI, completely changing front-end optimization. Previously, UI adjustment was "Edit Code → Save → Refresh → Check → Repeat." Now, it's "Select Element → Talk to AI → See Result." This isn't just faster; it changes the workflow.

When a title's color or spacing felt off, I didn't need to go back to the editor. I could select the element in the browser, give AI a command (e.g., "Darken this button's shadow"), and AI would generate the code and preview it immediately. This demonstrates the massive shift in development workflows: front-end optimization has moved from a "code loop" to a "real-time conversation." Chrome DevTools AI isn't just a tool; it represents a "paradigm shift"—from "Humans executing AI instructions" to "AI executing Human instructions." It was the most magical discovery of my project.

Chrome DevTools AI Real-Time Adjustment Demo
(Workflow: Select element → Tell AI "Darken shadow" → See immediate result)

Tool 8 & 9: Google Apps Script and Google Drive AI

When accessing Google Drive, I was pleasantly surprised to find it also had AI capabilities. Even more practical was AI teaching me to use Google Apps Script to automatically extract filenames and IDs from Drive folders. I combined screenshot → AI text extraction → semantic matching to let AI understand that "Solow-Swan Economic Convergence Report.pptx" should match the "solow-swan-model" entry, even if the names weren't identical. This automated workflow drastically improved efficiency for handling over 12 reports, completely eliminating manual copying.

Claude: The Cross-Functional Champion

Claude doesn't fit into just one category because it is cross-functional. It is currently the most robust AI model, usable in any scenario—whether for coding, logical discussion, or integrating into various tools.

Why Claude is Different

Of all AI tools, I use Claude the most. Whether for logical discussion or code debugging, its performance is the most stable. While it still "glitches" occasionally, the failure rate is much lower than others. After many comparisons, I found Claude significantly superior in: Logical Clarity—tracing reasoning paths is easiest; Debugging—precise issue localization; Low Hallucination Rate—more stable than Gemini 3; Code Generation—I believe it writes the best code of all AIs.

Claude loves to write code—often you have to restrain it: "Clarify the problem first, don't rush to code." But its coding ability is unmatched. More importantly, Claude isn't just a standalone chat AI; it integrates into every environment—Official Claude.ai, API, VS Code, Browser Extensions, etc. This cross-platform capability makes it a true "Cross-Functional Model." You need code? Claude. Logical discussion? Claude. Workflow integration? Claude. It is the strongest model for scenarios requiring high-quality reasoning and code generation. While Gemini 3 Pro wins on visuals, Claude delivers the most consistent performance in the core competencies of coding and logic.

Core Insight

True AI Capability: Recognizing Scenarios, Not Collecting Tools

The Key to Effective AI Tool Usage

Through hands-on experience with these tools, I've reached a core conclusion: The key to effective AI tool usage isn't about knowing every tool exists - it's about recognizing which scenario you're in and selecting the right tool for that context.

The three scenarios:
Need to generate websites? → Use One-Click Generators
Need to compare capabilities? → Use Model Arenas
Working within existing workflows? → Use Integrated Tools

And across all scenarios: Claude remains the strongest model for coding and logical reasoning, capable of integrating into any workflow.
